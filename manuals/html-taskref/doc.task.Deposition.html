    Prepare data for deposition¶

    The final stage of the crystallographic structure solution is the deposition of the final structure model, sequence, ligand data, and observations in the Protein Data Bank (PDB). The deposition is done by means of two mmCIF -formatted files: one containing atomic coordinates, sequence(s) and ligand descriptions, and one containing the observations (i.e., reflection dataset(s)). Both files are prepared by the Deposition task. The task examines the current project branch, finds the needful data and puts them in the resulting mmCIF files.

    Note

    The task does not actually deposit data in the PDB. It only prepares mmCIF files for deposition and acquires the PDB Validation report from wwPDB servers.

    After Deposition task finishes, inspect the PDB Validation Report, delivered by task. If you find that the structure is worth deposition, then:

    • Download the mmCIF files with final model (which also contains sequence(s) and ligand descriptions) and reflection data, using Download buttons in the task output page.

    • Start new deposition session at wwPDB Deposition Site (X-ray Deposition tutorial).

    Note

    The PDB accepts both merged (mandatory) and scaled unmerged (optional) reflection data. In CCP4 Cloud, scaled unmerged dataset is created by the Merging and Scaling with Aimless task. If this task was run in the current project branch, the scaled unmerged dataset will be found by the Deposition task automatically. In such case, the task produces mmCIF files for both the merged, and combined merged and scaled unmerged datasets. Of those, only one should be chosen for deposition at user’s discretion. Deposition of scaled unmerged data is encouraged in all cases.
