    CCP4 Cloud AlphaFold Utilities¶

    AlphaFold2 and RosettaFold predicted models are widely recognised as valuable structure templates for Molecular Replacement. This article outlines the integration of AlphaFold2 into CCP4 Cloud at various stages of the structure determination process.

    CCP4 Cloud tasks automatically recognize AlphaFold and RoseTTAFold models and convert their per-residue pLDDT scores (AlphaFold) or RMSD estimates (RoseTTAFold) into B-factors. The Import task provides a notice of this conversion in the Import Summary table, while other tasks, such as MrBump and MrParse, perform the conversion silently.

    Note

    It is essential to convert pLDDT scores and RMSD estimates, which are stored in the B-factor columns of predicted models, into actual B-factors. Without this conversion, the best parts of the model would receive the lowest weight in Molecular Replacement, significantly reducing the chances of obtaining a solution. This process is fully automated, so no manual intervention is required.

    pLDDT Cut

    AlphaFold generates a per-residue confidence estimate called pLDDT, which ranges from 0 to 100. This option sets the minimum threshold for selecting residues based on their pLDDT scores:

    • pLDDT > 90: Residues are expected to be positioned with high accuracy. These models are suitable for applications requiring high precision, such as characterizing binding sites.
    • 70 < pLDDT < 90: Residues are expected to be positioned reasonably well, generally providing a good backbone prediction.
    • 50 < pLDDT < 70: Residues are modeled with low confidence and should be treated with caution.
    • pLDDT < 50: Residues should not be interpreted. These scores strongly indicate disorder, suggesting that the region is either unstructured under physiological conditions or becomes structured only as part of a complex.

    Generating or Importing Predicted Models¶

    CCP4 Cloud not only supports the import of predicted models but also facilitates their generation for use in molecular replacement programmes like Phaser and Molrep.

    To generate models, CCP4 Cloud provides the following tasks:

    1. The Structure Prediction task, which employs AlphaFold-2 to generate structure models from a given sequence.

      After importing or generating a model, the Prepare MR Model(s) from Coordinate Data task prepares MR search models from the predicted structures.

    2. The Find and Prepare MR Models with MrParse task identifies structural homologues suitable for creating MR search models from the PDB and the AlphaFold2 Database. Relevant structures are selected based on sequence alignment using PHMMER.

    3. The Prepare MR Ensemble with CCP4mg task utilises MrBump to construct an MR Ensemble from structural homologues. This process involves alignment and truncation of homologous structures identified in the PDB and AlphaFold2 Database via sequence-based searches.

    4. The Prepare MR Ensemble from Sequence task uses MrBump to automatically prepare an MR Ensemble from a target sequence. This task includes identifying suitable structural homologues in the PDB and, optionally, the AlphaFold2 Database, trimming them, and superimposing the best candidates.

    Processing Predicted Models¶

    Inaccuracies in domain-domain orientations can limit the effectiveness of predicted models for MR. The Split MR Model with Slice-n-Dice task allows predicted models to be divided into distinct structural units, which can then be placed in an automated MR pipeline.

    Automated Molecular Replacement¶

    Automated Molecular Replacement pipelines search for homologous structures in the PDB and AlphaFold Database, generate suitable search models from the templates, perform molecular replacement, and validate the solutions with iterative rounds of restrained refinement.

    The following Automated Molecular Replacement pipelines in CCP4 Cloud support AlphaFold Database searches:

    1. MrBump’s model search applies alignment and truncation procedures using homologous structures from the PDB and AlphaFold Database.
    2. The MR with Model Splitting Using Slice-n-Dice task preprocesses AlphaFold2 and RosettaFold models by removing low-confidence regions and converting confidence scores into predicted B-factors. It can slice models into structural units using AlphaFold2’s predicted aligned error (PAE) or clustering algorithms for broader applicability. Slice-n-Dice addresses domain-domain orientation inaccuracies in predicted models.
    3. Arcimboldo Shredder automatically preprocesses AlphaFold or RoseTTAFold models by removing unstructured and disconnected regions and verifying MR solutions.

    Refinement¶

    Refinement optimises the agreement between the model and X-ray data as the final step in macromolecular crystal structure determination. For low-resolution data (below 3 Å), it is essential to utilise all available macromolecular information. The Low-Resolution Refinement with Lorestr task leverages the AlphaFold2 Database to identify homologues, enabling Lorestr to determine optimal refinement parameters.

    Automatic Workflows¶

    To demonstrate AlphaFold2’s integration into structure solution, CCP4 Cloud offers an automated project workflow: Molecular Replacement with AlphaFold Model. This workflow begins with data import and utilises the Structure Prediction and MR with Model Splitting Using Slice-n-Dice tasks to create the structure solution project automatically. Results can be refined further using individual tasks in manual mode for greater control over task parameters.
